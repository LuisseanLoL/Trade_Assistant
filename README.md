# 🤖 Trade Assistant (AI 量化交易智能体)

Trade Assistant 是一个基于大型语言模型（LLM）的自动化 A 股量化投研系统。本项目结合了**经典高阶量化指标（如 Hurst 指数、波动率分位、均值回归）与大模型的深度逻辑推理能力**，并通过首创的 **“漏斗式双模型降本架构”** ，实现了低成本的大规模市场扫盘与精准决策。

## ✨ 核心特性

* 🧠 **漏斗式双模型架构 (Cost-Effective Routing)**：使用免费/极低成本的模型（如 Gemini 3.1 Flash Lite）进行全市场海量粗筛。仅当发现“非观望”信号时，才触发推理能力更强、成本更高的付费模型（如 Gemini 3.1 Pro）进行二次深度复核，**在保证决策质量的同时将 API 成本降低 80% 以上**。
* 📊 **硬核量化指标支持**：集成 Baostock 与 AKShare，自动计算 52 周极值、PE/PB 历史分位、动量效应、布林带偏离度、RSI、ADX、ATR 真实波动率，以及高阶统计套利指标（赫斯特指数 Hurst、偏度、峰度）。
* 🌍 **宏观大盘风控**：自动拉取并融合上证指数（sh000001）的历史与盘中实时数据，计算大盘均线趋势、RSI 和 Z-Score。让 Agent 具备“全局视野”，防止在系统性大跌中盲目抄底。
* 💼 **自动化持仓管理**：通过本地 `portfolio.csv` 维护真实持仓。系统运行扫描时，真实持仓标的将享有“免检特权”，自动跳过初筛直接交由 Pro 高级模型进行深度体检。
* 🎲 **智能选股池切换**：支持“指定股票池”与“全市场随机抽盲盒”双模式无缝切换，且自动读取当日已生成的报表进行去重，实现高效的广撒网巡检。
* 🕒 **逻辑交易日管理**：自动识别物理时间，早上 9:00 前的运行将自动归属于“上一交易日盘后复盘”，数据归档井然有序。

## ⚙️ 系统架构与工作流

1. **数据抓取层**：通过 Baostock 获取前复权 K 线，AKShare 获取新浪财务基本面与同花顺机构预测，Playwright 无头浏览器绕过反爬抓取东财实时新闻。
2. **特征工程层**：计算长短周期均线系统，生成结构化的**量化策略信号矩阵**。
3. **Agent 决策层**：
* **Flash 初筛**：快速审阅特征与新闻，剔除无操作价值的“垃圾时间”标的。
* **Pro 精决**：对初筛通过的标的或本地持仓标的，综合宏观环境与个股特征，输出操作方向、建议仓位、目标价、止损价及深度逻辑分析。


4. **输出落地层**：生成人类可读的 Prompt 输入记录、LLM 原始分析文本，并追加清洗入当天的 `Daily Table_YYYY-MM-DD.csv` 结构化报表中。

## 🚀 快速开始

### 1. 环境依赖安装

本项目使用 Python 3.9+，请在终端执行以下命令安装依赖：

```bash
pip install pandas numpy baostock akshare beautifulsoup4 python-dotenv json-repair google-genai  openai efinance questionary

```

安装 Playwright 及内置浏览器（用于新闻无头抓取）：

```bash
pip install playwright
playwright install chromium

```

### 2. 环境密钥配置

在项目根目录创建 `.env` 文件，并填入您的 API 密钥及模型版本：

```env
gemini_api_key=你的_GEMINI_API_KEY
gemini_flash_model=gemini-3.1-flash-lite-preview
gemini_pro_model=gemini-3.1-pro-preview

```

### 3. 配置持仓与基础文件

* **`portfolio.csv`**（可选）：在根目录创建该文件管理真实持仓。表头必须为：`股票代码,持仓数量,持仓成本`。
* **`主板股票代码.csv`**（可选）：若开启随机扫盘模式，需在根目录提供包含 `股票代码` 列的全量 A 股代码表。

### 4. 运行 Agent

修改 `run_batch.py` 中的配置区（设定 `USE_RANDOM_BATCH` 等参数），然后在终端运行：

```bash
python run_batch.py

```

运行结束后，所有详细逻辑与聚合分析表格均可前往 `output/` 文件夹查看。

## 🗺️ 未来演进路线 (Roadmap)

本项目致力于向“完全自主进化的 AI 机构基金经理”迈进，接下来的改进计划如下：

* [ ] **1. 工程效率升级 (异步并发)**：引入多线程或异步协程 (`ThreadPoolExecutor`)，解决单只股票串行处理耗时过长的问题，将数百只股票的扫盘时间压缩至几分钟内。*(注：需平衡 API Token 频率限制及接口反爬策略)*
* [ ] **2. 认知架构升级 (交易记忆与反思)**：为 Agent 引入“短期记忆”。在新的决策循环中，传入前一交易日的判断逻辑，让模型在面对暴涨暴跌时产生“反思机制”，判定原逻辑是否已被破坏，增强在震荡市中的逻辑连贯性。
* [ ] **3. 架构演进 (Multi-Agent System)**：拆分单一 Agent 为“多智能体辩论系统”。分别由 Flash Lite 扮演“技术分析师”和“基本面研究员”给出独立研判，最终交由 Pro 模型扮演的“PM (投资经理)”结合风控做最终裁决。
* [ ] **4. 评价闭环 (自动归因与回测)**：建立 Feedback Loop，通过轻量级脚本自动读取历史 `Daily Table.csv` 进行胜率与涨跌幅回测验证。自动标记 `Bad Case` 和 `Good Case` 喂给模型进行错因分析，推动系统自我迭代寻找超额收益来源。
* [ ] **5. 模型竞技场**：引入多模型（如 Claude、DeepSeek）进行自动化胜率比较，动态调整最优的模型调用组合。
